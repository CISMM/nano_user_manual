Phantom and Magellan

If the Phantom or Magellan breaks, it is possible to prevent the NanoManipulator from trying to connect to them on startup. This will get rid of delays and error dialogs associated with not being able to connect to the broken equipment. See the Environment section for instructions on changing the environment variables which will enable or disable the Phantom and Magellan.

If the Phantom is not attached, you can still move the hand icon using Mouse Phantom emulation.

Slow response during replay

There are some instances where replaying a stream file can slow down considerably, causing the NanoManipulator to make large jumps in time, displaying changes infrequently. There are two things which can greatly reduced this problem. First, reduce the surface detail displayed during fast replays, by adjusting the Speed/Detail control. Second, reduce the number of points graphed by the Stripchart during modifications, by clicking the Axis Config button and reducing the Max pts. number.

ThermoMicroscopes SPMLab freezes

We have seen a rare problem which can cause the ThermoMicroscopes SPMLab application to stop responding. If you connect the NanoManipulator to and SPM, then disconnect by choosing File .. Close or opening a stream file, SPMLab may stop responding. If this happens, quit the NanoManipulator. SPMLab should start responding normally, and it has done so the few times we have seen this problem. If it does not, you may have to withdraw the scanning tip manually and force SPMLab to quit.
