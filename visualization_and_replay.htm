This tutorial will introduce basic features of the NanoManipulator, and cover visualization tools and the controls for replaying recorded experiments. 

Getting Started

Start the NanoManipulator. Double click the NanoManipulator icon on the desktop, or choose Start .. Program Files .. NanoManipulator .. NanoManipulator. When the program starts, you should hear the Magellan beep and the indicator light on the Phantom should change from orange or blinking green to solid green.

Open a static file. From the File menu,

 

choose Open Static Files... When the dialog opens, double-click on the folder named sample-data then choose the file adeno_spill_dna.tfr. 

When the file opens, the Surface View Window should show something like this:

[Click to see a full-sized image]

This image shows adenovirus, a virus which causes colds, but can also be used to deliver helpful DNA into cells during gene-therapy. The adenovirus on the lower right has ruptured, and its DNA is laying on the surface. 

To become familiar with the Phantom, try each of the modes described below. 

First, the green aim-line can help you determine the position of your hand in three dimensions, especially related to the position of the surface. Read about it in this short paragraph, then use the "back" button to return to this tutorial.

Change the function of the Phantom in two different ways. First, examine the main window below and observe all the buttons at the top. You can click on any of these buttons to change the function of the Phantom. Second, you can use the Magellan. This is the preferred method, because it is easier to use the Magellan in your non-dominant hand than it is to use the mouse and click on a button. The Magellan button which can activate each function labels a box surrounding the function. So button 1 activates Grab, button 5 activates Center, button 2 activates Position Light (and if you press it again it will activate Measure, then Scale Up, then Scale Down), button 6 activates Touch Stored. 

Buttons 3, 4, 7, and 8 are disabled, because these functions only work when connected to a live SPM, and not when looking at a static file or replaying an experiment. Their use is covered in the Manipulation Tutorial.

Click on the links to find out more information, then use the "back" button to return to this page.

Grab (Magellan button 1) - Press the Phantom button and move the surface. 

Center (Magellan button 5) - After you move the surface or the light, press Center to return to the default view. 

Position light (Magellan button 2) - Push the Phantom button, twist and rotate the pen to move the light.

Measure (Magellan button 2) - Dragging the red, yellow and blue lines lets you measure the size of surface features. 

Scale Up (Magellan button 2) - Enlarge your view. 

Scale Down (Magellan button 2) - Shrink your view. 

Touch Stored (Magellan button 6) - Push the Phantom button to feel the shape of the surface that you see. 

A few suggested tasks, which will help you become more comfortable with the Phantom.

Change the position of the light to best pick out the DNA on the substrate. 

Use the measure lines to find the width and height of an adenovirus. 

Position all three measure lines near one strand of DNA, then use scale-up to zoom in on the DNA. Now the measure lines are in easy reach to measure the height and width of a DNA strand. 

Save a Screen Image

Show others what you've done. Follow the Save Screen instructions to save a TIFF image. This format can be read by any Microsoft Office product, as well as most image viewers. 

For instance, in Microsoft Word, go to the Insert menu, and choose Picture .. From File. You should be able to see a preview of the TIFF file before you import it. 

Color Maps

Add impact to the surface view. We can use the color map to emphasize the features in the height plane, by showing the data from our static file with a color map as well. Follow the Color Map instructions, and choose the adeno_spill_dna.tfr as the color map plane, and choose the second color bar, blk-blue-cyan-wht, as the color map. You should see something like this (press Center if you need to):

[Click to see a full-sized image]

Interesting, but there is a slope to the substrate which we didn't notice before, which is emphasized by the color map. To eliminate this, we turn to Calculate Data Planes and create a Flattened plane. Click the Calculate Data Planes button to open the dialog, and type flat into the first entry widget, and press Enter. Close the window and choose flat as the Color map plane. Your view should change to this:

[Click to see a full-sized image]

Much better - the substrate is now nearly uniform in color, indicating that it is mapped to a single height. Now drag the arrows in the Colormap window to see how they affect the color map. In particular, if you drag the Color Max arrow down very close to the Color Min arrow, you should be able to highlight the DNA on the surface, like this:

To recover the default settings for the color map, click the Autoscale button. 

Automatic Lab Notebook

Next we'll start looking at a complete recorded experiment. 

Open an automatic lab notebook. Automatic lab notebooks are also called stream files. From the File menu, choose Open Stream File... When the dialog opens, you should already be inside the folder sample_data. Choose the file ohio-fibrin-9-26.nms, as shown, and click Open.

Replay Control

As soon as the stream file opens, it starts replaying. And the surface disappears! This is just a side-effect of the SPM scanning a different region of the surface than you were looking at before opening the stream file. Hit the Center button to get a view of the surface. 

Read the Replay Control instructions to find out how to control the replay of this stream file. 

The initial scan of the surface is usually uninteresting to watch. I happen to know the first scan of the surface takes 270 seconds, so type 300 into the Go to time: box and hit Enter to jump past the first scan. 

Also, we aren't limited to watching the experiment in real-time. Type 15 into the Speed (1,30): box to replay the stream file at 15 times faster than the original experiment. 

As you watch the progress of the experiment, press the Pause button at any time to freeze the action. Press Play to start again. Press Rewind to start over at the beginning. 

When the first modification takes place around time 1160, the Stripchart window will pop up. Read about it to find out what it is graphing and how to change what it is graphing. 

Z-scale

The fibrin fibers we are examining are fairly flat and a little bit hard to see. We can exaggerate their height to get a different picture of their shape. From the Setup menu, choose Height Plane..., and type 4 into the Z scale box. The heights measured by the SPM are now exaggerated by a factor of 4, making some surface features much easier to see. 

Rulergrid

Another tool which can make estimation, analysis, or presentation much easier is the Rulergrid. It simply overlays a grid on the surface. Choose Analysis .. Rulergrid and click Rulergrid Enabled to turn it on. A yellow grid will be overlaid on the surface. 

To illustrate how to use the grid for estimation, we're going to look at a specific time in this stream file. In the Replay Control window, Go to time 900, and Pause the stream file replay. 

Now let's adjust the rulergrid. Enter a Grid spacing of 200, and a Grid angle of 20. This means the yellow lines are spaced 200 nanometers apart, and they are displayed at a 20 degree angle from the edge of the scan. I used the Scale-up control to get this picture:

Examining the image, it appears that both fibers are nearly 200 nm wide. There is something else to consider, however, since we think fibrin fibers are cylinders, and an AFM tip will widen any feature it scans. This probably means the fibers are only 50-100 nm wide. 

If you want some more practice using the rulergrid or measure lines, find out how high one of the fibrin fibers is by placing one measure line on top, and another measure line beside a fiber. Then play the stream file to time 1900 and estimate the length of each of the modifications made using the rulergrid.
