Modification data is the information generated from a particular modification which has been recorded by the NanoManipulator. Saving the modification data allows you to store the information in a text format that KaleidaGraph, Excel, or another graphing/data processing program can read.

To save the modification data, first choose Save Modification Data from the File menu.

This dialog appears:

The Choose which modification to save: pop-up menu shows all modifications which have occurred so far, listed by the time they were started. If a modification from a stream file which you want to save does not appear in the list, click Cancel and use the Replay Control dialog to go to a time after the modification. 

Choose a modification from the pop up menu, and click Save.

Then this standard file dialog appears:

Choose a directory and a file name, and click Save. 

Modifications are stored in a tab-delimited text file, with a heading that identifies each column of data. The first four columns are: 
the time since the modification started
the path length the tip has traveled in nanometers
the x and y position of the tip in nanometers

The remaining columns are the data collected during the modification, which usually include the Topography, Internal Sensor, Lateral Force, and Z Piezo. Which data sets are collected can be adjusted using the Setup menu ... Datasets dialog.
